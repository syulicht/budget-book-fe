# Chrome Devtools Checker

## Overview

chrome-devtools MCPで、毎回の依頼内容に応じた確認項目を動的に作成して検証する。固定チェックリストは持たず、対象UIと要求から都度Inventoryを組み立てる。

## Workflow

1. Preflightを実行する。
2. Dynamic QA Inventoryを作る。
3. Inventoryに沿って検証を実行する。
4. 日本語フォーマットで結果を返す。

## Preflight

### 1) MCP接続確認
- `mcp__chrome-devtools__list_pages` を実行して接続を確認する。
- 接続不可なら、接続依頼を返して停止する。
- Playwrightなどの代替ツールには切り替えない。

### 2) アプリ到達性確認
- 既定URLは `http://localhost:5173` とする。
- 対象URLへ遷移し、到達可否を確認する。
- 到達不可なら `npm run dev` を実行して起動を試みる。
- 起動後、対象URL到達を再試行する。
- 再試行でも到達不可なら、観測したエラーを返して停止する。

### 3) 認証ハンドオフ
- Sign-in画面への遷移を検知したら、手動ログインを依頼する。
- ユーザーがログイン完了を通知したら、同じInventoryで検証を再開する。

## Dynamic QA Inventory

- 次の3入力から、今回の確認項目を毎回作る。
- ユーザー要求
- 現在のUIで観測できる操作要素（リンク、ボタン、フォーム、ナビゲーション）
- 変更影響が大きい導線（遷移、認証、送信、保存、破壊操作）
- 固定のページ名・文言・順序をSkill内にハードコードしない。

## 実行ルール

- 各確認項目で次を記録する。
- 実施した操作
- 観測結果（URL、表示状態、アクティブ状態、エラーメッセージ）
- 合否
- `list_console_messages` を使ってconsoleエラー有無を確認する。
- セッション/データ影響操作（ログアウト、削除など）がInventoryに含まれた場合は自動実行する。
- 推測で補完せず、観測事実のみ記録する。

## 返却フォーマット（日本語固定）

次の順で返す。

1. 実施項目一覧
2. 各項目の結果（成功/失敗 + 根拠）
3. 失敗箇所の詳細
4. 要手動対応（認証など）
5. 未実施項目と理由

## 停止条件

- MCP未接続
- 開発サーバー起動失敗
- 手動認証待ち
- 重大エラー発生（以降の検証継続で意味がない場合）
