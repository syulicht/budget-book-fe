# タスク完了時のチェックリスト

## コード変更後に実行すること

### 1. リント（必須）
```bash
npm run lint
```
- ESLintエラーがないことを確認
- 警告も可能な限り修正

### 2. 型チェック（必須）
ビルドコマンドで暗黙的に実行:
```bash
npm run build
```
- TypeScriptエラーがないことを確認

### 3. ローカルテスト（推奨）
```bash
npm run dev
```
- 開発サーバーで機能が正しく動作することを確認
- 認証フローが正常に動作することを確認

### 4. ビルド確認（推奨）
```bash
npm run build
npm run preview
```
- 本番ビルドが成功することを確認
- プレビューで動作確認

## デプロイ前のチェック
1. すべてのコミットがプッシュされているか
2. 環境変数が正しく設定されているか
3. ビルドが成功するか
4. S3バケットへのアクセス権限があるか

## コードレビュー時の確認事項
- 型安全性が保たれているか
- ESLintルールに準拠しているか
- 認証状態の管理が適切か
- エラーハンドリングが実装されているか
- セキュリティベストプラクティスに従っているか（特に認証関連）
